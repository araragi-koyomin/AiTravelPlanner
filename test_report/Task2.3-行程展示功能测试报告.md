## Bug 修复记录

### 测试执行过程中的 Bug 修复

#### 1. 日期格式匹配问题

**问题描述**:
- 测试用例中使用的日期格式与组件实际渲染的日期格式不匹配
- 例如：`2024-03-01` 与 `星期五 · 2024-03-01`

**根本原因**:
- 组件渲染时将日期与星期组合显示
- 测试用例期望精确匹配完整日期字符串

**修复方案**:
1. 使用正则表达式匹配日期部分
2. 将 `getByText('2024-03-01')` 改为 `getByText(/2024-03-01/)`

**修复的文件**:
- src/components/itinerary/ListView.test.tsx
- src/components/itinerary/TimelineView.test.tsx
- src/pages/ItineraryDetail.test.tsx
- src/integration/itineraryDetail.integration.test.tsx

#### 2. CSS 类名不匹配问题

**问题描述**:
- 测试用例期望的 CSS 类名与组件实际使用的类名不匹配
- 例如：`bg-blue-600` 与 `bg-primary-600`

**根本原因**:
- 组件使用 TailwindCSS 的 primary 颜色变量
- 测试用例硬编码了具体的颜色类名

**修复方案**:
1. 将测试用例中的类名改为组件实际使用的类名
2. 将 `toHaveClass('bg-blue-600')` 改为 `toHaveClass('bg-primary-600')`

**修复的文件**:
- src/pages/ItineraryDetail.test.tsx
- src/integration/itineraryDetail.integration.test.tsx

#### 3. 展开/折叠状态测试问题

**问题描述**:
- 组件默认展开第一天的行程
- 测试用例期望点击后展开，但实际已经展开

**根本原因**:
- 组件初始化时设置了 `expandedDays` 包含第一天
- 测试用例没有考虑到初始状态

**修复方案**:
1. 修改测试用例以匹配组件的实际行为
2. 测试点击后切换状态（从展开到折叠）

**修复的文件**:
- src/pages/ItineraryDetail.test.tsx
- src/integration/itineraryDetail.integration.test.tsx
