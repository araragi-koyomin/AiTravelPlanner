## 1. 测试前准备

### 1.1 环境准备

#### 开发环境
```bash
# 1. 确保已安装 Node.js 和 pnpm
node --version  # 应该 >= 18.0.0
pnpm --version  # 应该 >= 8.0.0

# 2. 安装项目依赖
pnpm install

# 3. 配置环境变量
# 复制 .env.example 到 .env 并填写必要的配置
cp .env.example .env

# 4. 配置 Supabase
# 确保 Supabase 项目已创建并配置完成
# 参考 docs/SUPABASE_SETUP.md

# 5. 配置第三方 API
# 在 .env 中配置智谱AI、科大讯飞、高德地图的 API Key
# 参考 docs/API_CONFIGURATION.md
```

#### 数据库准备
```bash
# 1. 确保数据库表已创建
# 在 Supabase Dashboard 中执行 SQL 脚本
# 参考 docs/DATABASE_DESIGN.md

# 2. 检查 RLS 策略
# 确保行级安全策略已正确配置

# 3. 准备测试数据
# 可以准备一些测试用户数据用于测试
```

## 3. 功能测试流程

### 3.1 整体测试流程

```
开始测试
  ↓
环境检查
  ↓
功能模块测试
  ├── 用户认证模块
  ├── 行程管理模块
  ├── 地图展示模块
  ├── 费用管理模块
  └── 语音输入模块
  ↓
集成测试
  ↓
性能测试
  ↓
兼容性测试
  ↓
安全测试
  ↓
记录测试结果
  ↓
问题报告
  ↓
测试完成
```

### 3.2 测试执行步骤

#### 步骤 1: 用户认证模块测试

1. **注册功能测试**
   - 访问注册页面 `/register`
   - 测试正常注册流程
   - 测试表单验证
   - 测试密码强度检查
   - 测试邮箱格式验证
   - 测试重复注册

2. **登录功能测试**
   - 访问登录页面 `/login`
   - 测试正常登录流程
   - 测试错误凭证登录
   - 测试记住我功能
   - 测试自动登录

3. **登出功能测试**
   - 测试正常登出
   - 测试登出后访问受保护路由
   - 测试登出后清除本地存储

#### 步骤 2: 行程管理模块测试

1. **创建行程测试**
   - 访问行程页面 `/itineraries`
   - 测试创建新行程
   - 测试表单验证
   - 测试必填字段验证

2. **查看行程测试**
   - 测试行程列表显示
   - 测试行程详情查看
   - 测试行程编辑

3. **删除行程测试**
   - 测试删除单个行程
   - 测试批量删除
   - 测试删除确认

#### 步骤 3: 地图展示模块测试

1. **地图加载测试**
   - 测试地图初始化
   - 测试地图缩放
   - 测试地图拖动

2. **标记点测试**
   - 测试添加标记点
   - 测试删除标记点
   - 测试标记点信息显示

#### 步骤 4: 费用管理模块测试

1. **费用记录测试**
   - 测试添加费用
   - 测试编辑费用
   - 测试删除费用

2. **费用统计测试**
   - 测试费用汇总
   - 测试费用分类统计
   - 测试费用图表显示

#### 步骤 5: 语音输入模块测试

1. **语音识别测试**
   - 测试语音输入启动
   - 测试语音识别准确性
   - 测试语音输入停止

2. **语音输入集成测试**
   - 测试语音输入到表单
   - 测试语音输入错误处理

## 6. 问题报告流程

### 6.1 问题发现

当测试过程中发现问题时，需要记录以下信息：

1. **问题基本信息**
   - 问题标题
   - 发现日期
   - 发现人员
   - 严重程度

2. **问题详细描述**
   - 问题描述
   - 复现步骤
   - 预期结果
   - 实际结果

3. **环境信息**
   - 浏览器版本
   - 操作系统
   - 应用版本

4. **附件**
   - 截图
   - 录屏
   - 日志文件

### 6.2 问题严重程度分类

| 严重程度 | 定义 | 示例 |
|---------|------|------|
| 严重 | 系统崩溃、数据丢失、安全漏洞 | 登录后无法访问任何页面 |
| 高 | 主要功能无法使用 | 无法注册新用户 |
| 中 | 部分功能受影响 | 密码强度提示不准确 |
| 低 | 界面显示问题、文字错误 | 按钮文字显示不正确 |

### 6.3 问题报告模板

```markdown
# 问题报告

## 问题描述
[简要描述问题]

## 复现步骤
1. [步骤1]
2. [步骤2]
3. [步骤3]

## 预期结果
[描述预期结果]

## 实际结果
[描述实际结果]

## 环境信息
- 浏览器: [浏览器名称和版本]
- 操作系统: [操作系统名称和版本]
- 应用版本: [应用版本号]

## 严重程度
[严重/高/中/低]

## 附件
- [截图]
- [录屏]
- [日志]
```

### 6.4 问题跟踪

1. **问题记录**
   - 在测试记录表格中记录问题
   - 为每个问题分配唯一ID

2. **问题跟踪**
   - 使用 GitHub Issues 或其他问题跟踪工具
   - 定期更新问题状态

3. **问题验证**
   - 修复后重新测试
   - 确认问题已解决
